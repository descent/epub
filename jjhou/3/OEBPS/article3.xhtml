涼風起天末

秋收冬藏。秋天是驗收成果的季節, 冬天則應該圍爐怡情, 養精蓄銳, 以待來年再出發。呵, 農業社會分明的節氣, 多麼令工業社會中繁忙的我們嚮往。其實我們也並沒有完全忘記節氣, 你看, 12 月來了, 你的心中開始盤算耶誕、新年、尾牙、春節、春酒..., 想著想著就到了明年的青年節, 心情不自主地輕鬆了起來。

突然間想到一個笑話, 講給你聽。一個農夫在星期一清晨四點鐘叫醒他還在睡夢中的兩個兒子說: 「看看你們, 再兩天就是星期三, 一個星期就過掉了一半, 還不趕快起來工作！」我曾以此笑話勉勵我的學生。

這個世道未免飆得太快了些, 飆得咱們資訊人累得半死。去年底 Win95 出來, 把大家搞得人仰馬翻; 今年 Web 當紅, 所有人統統擠上資訊高速公路; API 不斷暴漲, MFC 不斷增胖; C++ 還沒精通, 又搞出一個Java。哎, 資訊人真的快累癱了。

歲末, 就讓咱們輕鬆一下吧。

●沛然不得不發

我曾回答為數不少的讀者來自 Email 的詢問, 他們問我如何寫作, 問我如何把持方向, 問我有關於「侯氏風格」的形成。他們說已經習慣我的筆法, 宛如上癮般。他們問我的地址, 然後熱心地把作品寄給我, 「請我指導」。

在快速更迭一如白浪掏沙的資訊界以及資訊傳播界, 一個寫作將滿六整年的人, 大概也算得上是老兵了吧。讓我藉這個機會說點自己對技術寫作的看法。

首先我要說, 你, 拿起筆, 應該是因為對某個主題有了沛然不得不發的燥熱感, 不行之為文昭告天下, 無法抒解你滿腔的快樂與興奮, 然後你才拿起筆。這種燥熱感也許就像, 唔, 如果你讀過倚天屠龍記的話, 就像張無忌在乾坤一氣袋中受到九陽神功和幻陰指兩種內力的激盪, 快要爆炸了一般。在這種情況下拿筆, 就動機與時機而言, 比起嗅覺靈敏的市場慣性作家, 你已經在出發點上校正了你的起跑架。

Kraig Brockschmidt（Inside OLE 一書作者）曾經在他的自序中講過一句耐人尋味的話:

It has been said that authors write books not so that they will be understood, but so that they themselves understand.

我請教了不少朋友, 對這句話的看法莫衷一是。我個人的解釋是:

有人這麼說: 作者之所以寫書, 並不是因為這麼一來他們的想法可以被大眾了解, 而是因為他們自己可以因此對知識獲得更多的了解。

「因此對知識獲得更多的了解」？我的朋友對此有些微言。他認為作者的意思是: 因為寫書而逐漸地增強了自己的功力。如果再強硬地推導一下, 就變成了: 「因為我想了解這本書, 所以我來寫這本書」。這就帶來一種「作者與讀者共成長」的尷尬感覺, 產生出「作者並不是因為『會』而寫這本書, 而是為了『想要會』而寫這本書」的印象。

「沛然不得不發地寫作」和「在寫作過程中獲得更多的啟發」其實是不悖離的。作者必定要有相當程度的技術水準與見解, 才擬得出見骨見肉的題目和大綱。而一個負責任的作者, 一定會想盡辦法克服寫作過程中遇到的窒礙, 自然而然也就提升了自己的水平。

如果按我自己的寫作理念, 我要把 Kraig Brockschmidt 的那一句話改為:

I write books not only so that I will be understood, but so that I myself understand.

我之所以寫書, 並不只是因為這麼一來我的想法可以被大眾了解, 也因為我可以因此對於知識獲得更多的了解。

是的, 所有出於我的筆鋒之下的東西, 我自己獲益最豐。

●兩個基本

談談技術書籍寫作的基本技能。我認為一個作家, 應該具備兩項基本技能。第一項基本技能是中文輸入。有了這一能力, 你才可以自由自在十分方便地修改文字、調整段落, 也可以省掉許許多多因為「手民誤植」、校稿不週而造成的文字錯誤。老實說, 我無法想像, 如何在稿紙上以手寫方式完成技術書籍的創作（翻譯倒還有一點點可能）。援筆立就, 倚馬可待, 是一種神話, 遙不可及 -- 至少對我們凡人遙不可及。林清玄先生稱自己寫稿幾乎不加修改, 一萬個字改不超過五個字, 這種神奇的文字能力真是令人嘆為觀止。

第二項基本技能是排版。我所指的排版, 包括熟悉一套排版軟體, 和一套能夠表現你的構圖 -- 流程圖、示意圖等等 -- 的繪圖軟體。根據我的經驗, 書籍寫好了之後, 正是痛苦的開始, 因為你開始要面對排版人員和美工人員了。技術書籍中繁複的粗體字、斜體字、大中小標題、圖片的安排、字形的選擇、行距、段距、留白, 在在影響閱讀舒適感和容易理解度。這些散佈全書各處的瑣碎細節, 如果不是有默契良好的排版人員配合, 溝通的過程將是體力和耐力的大考驗, 是十分令人氣餒和挫折的過程。噢不, 沒有理由怪排版和美工, 他們不了解你書中所寫的東西, 也不了解你的想法與個人好惡, 自然需要一再溝通, 「一試再試做不成, 再試一下」。

為了能夠充份並完全地掌握品質, 擁有排版能力是一位作家應該有的基本能力。一定有很多人不同意這種說法, 認為作家應該把心力全部放在技術的研究和文字的斟酌上。我完全同意 -- 如果確能如此。事實上, 數年以前, 大家不也認為 中文輸入是輸入員的事？現在大家不也認為中文輸入是每個技術作家的基本技能？我相信這樣的觀念轉移也會發生在排版身上。我自己, 已經無法想像把作品交給別人排版的災難。

海明威曾經說: 一個好的文學家, 必須有一個不幸的童年 。我們很慶幸我們沒有想當文學家, 所以我們不必有一個不幸的童年。如果你希望你的作品鶴立雞群, 你必須有自己的特色。不幸的童年可以換取敏銳易感的特色, 但很不幸的是你我的童年已經過去了。然而, 我們可以用更快樂的方式培養諸如音樂欣賞、美術欣賞、國學能力、文學素養、其他專業領域等等特色。這些素養偶爾而適當地見諸於你的文字中, 就會使你的作品更豐富。時時記下遇到的好句子, 在最適當的地點用上它, 更有畫龍點睛之妙。

很多外文電腦書籍作者拿的是粒子物理的學位, 因為巧合還是因為某種訓練上的關連, 不得而知。他們在技術文字中把某些現象, 以物理來解釋或譬喻, 或寫些有關於物理應用的範例, 讓人讀來興致盎然。

我曾經在無責任書評中舉過一個例子, Charles Petzold 在一篇名為 Working with Lines And Pens Under Windows NT （原載於 PC Magazine）的文章內, 從他的幾何學啟蒙課程開始, 談到歐幾里得, 談到愛因斯坦的光與重力的影響, 然後ㄠ到電腦螢幕和印表機的輸出, 笛卡兒的解析幾何, 再ㄠ回電腦繪圖以及Windows NT 上繪點繪線的技術。這是我看過最精采的一次柳暗花明的技巧示範。

看看下面這一段由 Kaare Christian 所寫的 Making Exceptions with C++ （原載於 PC Magazine）中的第一段:

語言, 蝸牛, 猴子和人類, 這之中唯一的共同點是進化。C++, 就像英文一樣, 也是一個經過進化的語言。今天大部份我們所使用的 C++ 語言多多少少比標準化制定委員會的完整版本少了某些優點, 但是自八十年代未期以來所使用的C++ 型式, 的確已經有更多不可忽視的演化。例如...

再看看 Kaare Christian 所寫的 C++ Template Technology （原載於 PC Magazine）中的第一段:

無性生殖並不只存在於遺傳工程上, 對程式員而言它也是一個由來已久的動作。過去, 我們只不過是以一個簡單而基本的工具, 也就是一個文字編輯器, 來重製我們的程式碼。今天, C++ 提供給我們一個更好的繁殖方法 : template。

幾個看似隨性的譬喻, 三言兩語, 文章就活絡了起來, 也達到了吸引讀者注意的目的。

要贏得別人的信賴並不容易。贏得讀者的信賴是技術文字工作者的第一目標。擴充自己的視野, 使自己的作品豐富, 帶給讀者閱讀樂趣, 是技術文字工作者的最高層次。

●志村大爆笑

10/27 日的一則影藝新聞吸引了我的注意力, 算是一則好消息:

志村健死了？這個謠言從七月起就在日本廣為流傳。志村健最近在富士電視台的「大今天」節目中現身, 否認這個謠言。他說: 「我活得好好的。」

我第一次看志村大爆笑（一個電視節目）, 是在東京。日本話完全聽不懂, 但是從頭狂笑到尾。（侯捷唯一會講的兩句日本話是「現在幾點了」和「我是交通大學的學生」, 這是大二修了一學期日語的成果。）

我第二次看志村大爆笑, 是在臺灣, 在一本 Windows NT 書籍的中譯本上看到。形雖異, 實相同, 也讓我從頭狂笑到尾。這一段奇歷以及大爆笑的精采片段, 記錄於無責任書評２之中。

日本的志村健, 我希望他長命百歲, 繼續帶給人們歡笑。臺灣的志村健, 我希望最好都消失不見, 我們的電腦書的前途才有可能長命百歲。

●兩個非常

十月初應邀對台大資訊系新生做一場演講。他們為我量身定製了一個「資訊未來」的題目。我說這件西裝太大, 適合給 Bill Gates 或施振榮先生來穿, 我穿小兩號的。所以我給自己定了一個題目: 「作為一個資訊人」。
題目很浮泛, 焦點很模糊, 大家很輕鬆。

我談到資訊界的兩個非常。第一個非常是「非常幸福」。資訊工業是一個未臻成熟的工業, 所以不斷推陳出新, 就業機會與創業機會很多很多, 對於高技術高學歷的人才需求量非常大。就「錢」途而言一片光明。

非常幸福其實不只於「錢」途。能夠跟著世界脈動, 個人隨之成長的那種機會與那種感覺, 才真正是人生非常幸福的一件事兒。報刊雜誌上成天講 Web, 我們知道; 風行世界的 Internet, 我們輕易能夠使用; 桌上排版、傳真、簡報, 我們也可以輕輕鬆鬆把它融入到生活之中。為什麼我們比別人更幸福？因為我們就在這個環境裡面。不管在軟硬體環境上或學習環境上, 我們都有探索不完的知識, 資源嘛則是垂手可得。

第二個非常是「非常辛苦」。正因為資訊界不斷推陳出新, 所以我們需要不斷學習, 不斷競爭。我住的大樓裡有許多工研院和科學園區的研究員與工程師, 有一次我在電梯內遇到一位鄰居, 他老兄安全帽一脫, 眼袋下垂, 嘴角鬆弛, 整個人攤在電梯牆壁上, 對著陌生人（我）說: 「哎, 累死人了」, 然後又喃喃自語不知道說些什麼。如今我還偶而會在電梯間遇到他, 沒有一次他不咧嘴一笑（比哭還難看）地說: 「哎, 累死人了」。這不是個案或特例, 我的朋友們, 包括以前我自己, 不管是有酬加班也好, 無酬加班也好, 志願加班也好, 強迫加班也好, 加班平常得就像喝牛奶一樣, 只差沒有當水喝, 天天喝。

但是我們痛苦嗎？不, 我們不痛苦, 辛苦不等於痛苦。痛苦的人都離開了, 還沒離開的也大概醞釀要離開了。好逸惡勞是人的天性, 可人性之中還有一些其他因素會蓋過這個天性, 包括成就感啦, 事業心啦、上進心啦、求知欲啦。

●輸在起跑點上？

我發現, 在資訊系裡面特別容易出現起跑點極端懸殊的例子。有的同學早在高中, 或國中, 或甚至小學就開始接觸電腦, 可能在進入大學時就已經會寫 Windows 程式了, 或者對 C/C++ 已經很熟了, 或者常常上網, 甚至架設過站台。另一些同學沒有這麼早的啟發機會, 於是一開學他們發現「我已經輸了」。距離是如此懸殊, 二枚腰逆轉的機率看起來不大。

啊, 我要警告那些佔有優勢的同學, 要謙虛, 不要狂妄。在我看來, 你們所佔的優勢, 不過就是對 PC 環境以及對程式設計的熟悉, 這兩者在計算機科學裡只不過是滄海一粟, 沒有什麼好自以為了不起的。記住, 這是一個沒有倫理的領域 -- 我的意思是, 沒有誰能夠永遠作大哥。

對於那些還沒起跑就已經落後的同學, 我要你們想想龜兔賽跑的故事。如果你們問我, 那隻兔子永遠不睡覺怎麼辦, 讓我提醒你, 你其實並不是那隻烏龜。再者, 如果那隻兔子很勇猛但不持久, 而你雖不勇猛卻很持久（不要笑）, 你終於還是會贏的。人生不是五年、七年的事, 人生是五十年、七十年的事。再者, 如果那隻兔子很勇猛又持久, 那...那你輸給他也是應該的嘛！你還可以贏很多人啊。

我知道聽我演講的這個班上, 除了 50 個聯考名額, 另外還有總共近 20 名的高中成績優異甄選入學者和國際奧林匹克數學競賽保送者。一位已經十分優秀的同學憂心忡忡地問我有關於心理調適的問題, 問我讀書的法門, 問我國外期刊怎麼訂怎麼讀。我真的感受他的焦灼, 儘管他已是如此優秀, 而迎新舞會都還沒有舉辦。

自己和自己比！今天的自己比昨天的好, 明天的自己要更勝今天, 這樣就能活出自己的天空。
